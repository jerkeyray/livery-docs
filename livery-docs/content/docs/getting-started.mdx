## Before you install anything

There is no public npm release yet. The package names in the monorepo are real, but `bun add @jerkeyray/react` currently points at a package that npm cannot resolve.

For now, work from the repository:

```sh
git clone https://github.com/jerkeyray/livery.git
cd livery
bun install
bun run build
```

The playground is the fastest way to try the language from the repository:

```sh
bun run dev
```

## Package map

| Package | What it owns |
| --- | --- |
| `@jerkeyray/core` | Language, compiler, pinboard solver, validation, timelines, and SVG/JSON export |
| `@jerkeyray/web` | Framework-independent retained browser runtime |
| `@jerkeyray/react` | React components for static and streamed source |
| `@jerkeyray/export-node` | PNG rasterization for Node and Bun |
| `@jerkeyray/cli` | Validation and SVG, PNG, or JSON output from the command line |
| `@jerkeyray/layout-elk` | Optional adapter for complex legacy graph layout |

Once the packages are published, most React applications will only need `@jerkeyray/react` and its stylesheet. Until then, treat the examples below as the stable API direction rather than an installation promise.

## Write a first figure

Create `request.livery`:

```text
figure request("Request path") {
  browser = browser("Browser")
  gateway = api("Gateway")
  worker = worker("Worker")

  submit = browser.right -> gateway.left("POST /jobs")
  dispatch = gateway.right -> worker.left("dispatch")

  row(browser, gateway, worker, gap: xl)
}
```

The syntax is intentionally boring:

1. Declare one figure.
2. Bind components to stable names.
3. Connect named anchors.
4. Describe the macro layout.

The solver handles the rest.

## Render from the core API

```ts
import { render } from "@jerkeyray/core";

const result = render(source, { width: 720 });

if (!result.svg) {
  console.error(result.diagnostics);
  return;
}

document.querySelector("#figure")!.innerHTML = result.svg;
```

Do not test only `diagnostics.length`. Warnings may exist alongside a valid scene. The useful boundary is whether the output you asked for exists.

## Render in React

```tsx
import { LiveryVisual } from "@jerkeyray/react";
import "@jerkeyray/react/styles.css";

export function Figure({ source }: { source: string }) {
  return <LiveryVisual source={source} responsive />;
}
```

`LiveryVisual` owns the browser runtime instance, updates it when `source` changes, and cleans it up when the component unmounts.

## What failure looks like

Invalid source does not produce a half-valid SVG. The result contains diagnostics with a code, severity, message, and source span when one is available:

```ts
for (const item of result.diagnostics) {
  console.error(item.code, item.message, item.span);
}
```

That is not just nicer error reporting. It is what makes an agent repair loop possible without parsing error prose or guessing which part failed.
