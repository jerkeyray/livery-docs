---
title: React and streaming
description: Render static source, retain valid scenes during streaming, and control timelines.
---

There are two React components. Use `LiveryVisual` when you control the complete source. Use `LiveryChatVisual` when the source arrives token by token and may be temporarily invalid.

## Static or controlled source

```tsx
import { LiveryVisual } from "@jerkeyray/react";
import "@jerkeyray/react/styles.css";

export function ArchitectureFigure({ source }: { source: string }) {
  return (
    <LiveryVisual
      source={source}
      responsive
      onDiagnostics={(diagnostics) => {
        diagnostics.forEach((item) => console.warn(item.code, item.message));
      }}
    />
  );
}
```

`responsive` observes the container width and resolves the scene again when that width changes. Pass an explicit `width` when rendering in a fixed surface or an environment without `ResizeObserver`.

## Streamed source

Generated source is incomplete for most of its life. A missing closing brace halfway through a stream is not a final failure; it is just Tuesday.

`LiveryChatVisual` keeps the last valid scene visible while new source is still arriving:

```tsx
import { LiveryChatVisual } from "@jerkeyray/react";
import "@jerkeyray/react/styles.css";

export function AssistantVisual({
  source,
  streaming,
}: {
  source: string;
  streaming: boolean;
}) {
  return (
    <LiveryChatVisual
      source={source}
      streaming={streaming}
      compileDelay={80}
      fallback={<p>The visual could not be rendered.</p>}
      onRevision={(revision) => {
        console.log(revision.status, revision.diagnostics);
      }}
    />
  );
}
```

The fallback appears only after streaming has finished and the final source is invalid. During the stream, the component prefers the last valid scene.

## Revision states

Every revision has one of four statuses:

| Status | Meaning |
| --- | --- |
| `empty` | There is no source and no retained scene |
| `ready` | The current source produced a valid document and scene |
| `retained` | The current source is invalid or incomplete, so the previous valid scene remains visible |
| `invalid` | No valid scene is available for the current source |

Use `onRevision` for product telemetry and generation quality. Do not scrape visible error text from the component.

## Timeline controls

If the figure contains a timeline, `LiveryChatVisual` can expose accessible state controls:

```tsx
<LiveryChatVisual
  source={source}
  streaming={false}
  timeline="checkout"
  timelineControls="auto"
/>
```

`timelineControls` accepts:

- `auto` — show controls when more than one state is available
- `always` — show controls whenever the selected timeline has states
- `hidden` — render the selected state without controls

The controls support arrow keys, Home, and End. State changes are announced through an ARIA live region.

## Controlled timeline state

```tsx
const [state, setState] = useState<string>();

<LiveryChatVisual
  source={source}
  streaming={false}
  timeline="checkout"
  state={state}
  onStateChange={setState}
/>
```

Leave `state` undefined for internal control. Once you pass it, your application owns the selected state.

## Lower-level access

`LiveryVisual` exposes the mounted browser instance through `onRender`:

```tsx
<LiveryVisual
  source={source}
  onRender={(instance) => {
    instance.setState("complete");
  }}
/>
```

Prefer props for normal React state. Reach for the instance when integrating with an external controller or a non-React event source.
