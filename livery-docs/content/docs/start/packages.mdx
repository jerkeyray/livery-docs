---
title: Choose an entry point
description: Install one package and import only the runtime boundary you need.
---

| Public entry point | Choose it when… |
| --- | --- |
| `liveryscript` | You need compilation, solving, validation, SVG/JSON, themes, catalogs, or agent helpers without a UI framework. |
| `liveryscript/web` | You mount and update visuals directly in the browser. |
| `liveryscript/react` | You render complete or streamed source in React. |
| `liveryscript/node` | Node or Bun must rasterize validated SVG to PNG. |
| `livery` executable | A shell or CI job validates and exports `.livery` files. |

All entry points ship in the single LiveryScript distribution. Most React products use `/react` plus `liveryscript/styles.css`. Server-only systems use the root and optionally `/node`. Do not import Web internals from a React application unless you need direct instance control.

Core exposes compatibility APIs for the earlier `flow` artifact pipeline. New work should use `figure`, `render`, `exportVisual`, `VisualDocument`, and `BoardScene`.
