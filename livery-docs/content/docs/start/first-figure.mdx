---
title: Your first figure
description: Author stable elements, connect anchors, choose a layout, and inspect diagnostics.
---

Create `request.livery`:

```livery
figure request("Request path") {
  client = browser("Browser")
  api = service("Orders API")
  orders = database("Orders")

  send = client.right -> api.left("POST /orders")
  store = api.right -> orders.left("write", tone: success)

  row(client, api, orders, gap: xl)
}
```

The bindings are stable identities. Anchors make direction explicit. `row` gives the solver composition rather than board coordinates.

Create `render.mts` beside it in the repository root:

```ts
import { render } from "./packages/liveryscript/dist/index.mjs";

const source = await Bun.file("request.livery").text();

const result = render(source, { width: 720 });
if (!result.svg) {
  console.error(result.diagnostics);
  process.exit(1);
}

await Bun.write("request.svg", result.svg);
console.log("wrote request.svg");
```

Build and run the complete example:

```sh
bun run build
bun render.mts
bun packages/liveryscript/dist/cli.mjs request.livery --output request-cli.svg
```

Both commands compile and validate the same source. The script exercises the public root API; the second command exercises the public `livery` executable before it is installed from npm.

Warnings may coexist with valid output, so check the output you requested rather than `diagnostics.length`.
