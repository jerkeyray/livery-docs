The architecture is organized around one rule: an invalid scene does not reach a renderer.

```text
source
  → tokenizer and parser
  → semantic validation and component expansion
  → VisualDocument
  → pinboard candidate solving
  → geometry and resource validation
  → BoardScene
  → SVG, browser runtime, React, JSON, PNG, or CLI
```
