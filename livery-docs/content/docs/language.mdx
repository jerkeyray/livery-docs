Livery is a bounded visual language. It is programmable enough to reuse structure, but deliberately missing the things that turn generated code into a security problem: no JavaScript, no I/O, no mutation, no recursion, and no unbounded loops.

## The mental model

A program contains reusable components followed by one `figure`. Inside the figure, you bind visual elements, connect them, and describe how they should be composed.

```text
figure checkout("Checkout request") {
  customer = person("Customer")
  api = service("Checkout API")
  orders = database("Orders")

  submit = customer.right -> api.left("submit order")
  persist = api.bottom -> orders.top("persist", tone: success)

  grid(customer, api, orders, columns: 2, gap: xl)
}
```

Bindings such as `customer` and `persist` are not disposable variable names. They become stable identities used by connectors, timelines, diagnostics, and retained browser updates. Give them names worth keeping.

## Components

The standard library covers the common shapes used in technical explanations: people, browsers, services, APIs, workers, databases, queues, streams, agents, tools, models, files, documents, callouts, and more.

You can call them directly:

```text
client = browser("Browser")
api = service("Orders API")
db = database("Postgres")
```

Or use the explicit library namespace:

```text
agent = lib.agent(label: "Research agent")
search = lib.tool(label: "Search")
model = lib.model(label: "Reasoning model")
```

The library components are conveniences built on public visual primitives. They are not secret compiler entity types.

## Custom components

Components let you reuse a visual structure without copying it:

```text
component Pair(left: string, right: string = "Worker") {
  first = box(left)
  second = box(right)

  return row(gap: lg) {
    first
    second
  }
}
```

Parameters support `string`, `number`, `boolean`, and `tone`. Expansion is bounded, so a component cannot recursively create an unbounded scene.

## Connectors and anchors

Connectors join explicit anchors: `top`, `right`, `bottom`, `left`, or `center`.

```text
request = client.right -> api.left("request")
write = api.bottom -> db.top("write", tone: success)
```

Prefer the anchor that explains the intended flow. The router may choose the exact path, but the relationship remains stable and readable.

The functional form is useful when named arguments read better:

```text
query = connect(agent.right, search.left, label: "query", variant: async)
```

## Layout

Use macro layout for the board:

- `row` for left-to-right sequences
- `column` for top-to-bottom sequences
- `grid` for repeated peers
- `stack` for layered composition
- `overlay` for deliberate overlap
- `canvas` for bounded local drawing

```text
row(customer, api, payment, orders, gap: xl)
```

Spacing tokens such as `sm`, `md`, `lg`, and `xl` are usually a better choice than raw numbers. They preserve the relationship when the theme or viewport changes.

Constraints such as `align`, `distribute`, `inside`, and `near` can refine composition without turning the whole figure into coordinates.

## Canvas

Sometimes a figure is not a graph. A chart, waveform, mechanism, or scientific sketch needs local coordinates. Put those coordinates inside a bounded canvas:

```text
figure plot("Signal") {
  art = canvas(width: 240, height: 140) {
    axis = line(x1: 20, y1: 120, x2: 220, y2: 120)
    curve = path(
      d: "M 20 110 C 80 100, 140 30, 220 20",
      stroke: "#2563eb",
      fill: "none"
    )
    label = text("signal", x: 150, y: 30)
  }
}
```

The coordinates are local to that canvas. They do not become a back door for positioning the whole board by hand.

## Timelines

Timelines change presentation while keeping the solved geometry stable:

```text
timeline checkout {
  state request {
    show(customer, api)
    trace(submit)
  }

  state complete {
    show(orders)
    trace(persist)
    set(persist, tone: success)
  }

  transition request -> complete(duration: normal)
}
```

The supported operations are `show`, `hide`, `focus`, `trace`, and `set`. Use a timeline when state changes add meaning. Do not add one just because motion looks nice.

## Rules worth remembering

- Keep labels short. Layout cannot rescue a paragraph inside a node.
- Bind anything you need to connect, animate, inspect, or repair.
- Prefer macro layout and spacing tokens over board-level coordinates.
- Use one figure per generated response.
- Treat every diagnostic as real. Unknown calls and properties are not silently ignored.

For agent-generated source, do not copy this page into a prompt. Use the release-matched [agent guide](/docs/agents) generated by the library.
