# Security policy

## Reporting a vulnerability

Do not open a public issue for an undisclosed vulnerability.

- For this documentation site, Studio, deployment configuration, or hosted API routes, use a [private security advisory](https://github.com/jerkeyray/livery-docs/security/advisories/new).
- For the Livery compiler, runtime, exporters, or CLI, report it in the [main Livery repository](https://github.com/jerkeyray/livery/security/advisories/new).

Include the affected revision, impact, reproduction steps, and any known mitigation. Do not include production credentials, raw client addresses, private prompts, or other sensitive user data.

The current alpha receives security fixes on the latest `main` revision. Hosted Studio security boundaries and rate limits are described in [Security and limits](https://livery.jerkeyray.com/docs/operations/security).
